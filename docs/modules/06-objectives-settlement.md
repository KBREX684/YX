版本：v0.3.2

创建日期：2026-05-14

## 模块定位

副本目标与结算模块负责管理逃离、击杀、收容三种完成方式，以及它们对应的奖励、后果和长期影响。

三种目标不是简单难度选项，而是玩家理解副本深度的体现：逃离代表活下来，击杀代表反制，收容代表完整理解并控制异常。

## 关联 Design Pillars

- **Pillar 1（看不见但可学习的怪物）：** 收容奖励最高，正是因为它需要最深的规则理解。结算系统在叙事条目维度对理解深度给予奖励，强化"学习是有价值的"。
- **Pillar 2（任何变强都同时意味着变危险）：** 收容过程风险最高，收容失败（错误收容）惩罚也最重。高奖励必须对应高风险，不存在无代价的最优路径。

## 设计目标

- 让三种目标在同一个副本内成立。
- 让不同目标对应不同风险、奖励和叙事后果。
- 鼓励玩家先逃离、再击杀、最终尝试收容。
- 保证玩家不必每次都完美收容，也能推进游戏。
- 通过三维奖励结构区分三条路径的价值，防止目标塌缩。
- 通过结算把副本内选择带回基地养成。

## 核心功能

### 三维奖励结构

奖励不再是单一"素材好坏"，而是三个维度独立计算，每条目标路径在不同维度上有优势：

**维度 ①：基础素材量（Survival Resources）**

- 玩家从副本携带回的生存物资、养成素材的数量。
- 逃离 = 高（时间和精力集中在搜刮上，没有被消耗于击杀/收容操作）。
- 击杀 = 中（击杀过程消耗部分物资，但怪物被处置后部分区域可安全搜刮）。
- 收容 = 低（时间和注意力集中在线索和仪式上，搜刮机会减少）。

**维度 ②：原形品质与路线适配（Origin Quality & Affinity）**

- 产出原形的品质等级、稳定度，以及与特定养成路线的亲和度。
- 逃离 = 无（不获得原形，也不获得原形残片）。
- 击杀 = 中品质、低稳定度（偏战斗/恐怖向，适合恐怖化路线）。
- 收容 = 高品质、中/高稳定度（多路线选择，适配性强）。

**维度 ③：叙事与情报条目（Narrative & Intel）**

- 解锁的怪物档案页数、世界观情报条目、叙事记录完整度。
- 逃离 = 低（只有副本基础信息，无法了解怪物真实规则）。
- 击杀 = 中（解锁"感知方式"和"弱点"，但来源真相损坏）。
- 收容 = 高（完整档案、世界观情报、旧机构关联线索）。

**三路径奖励对比表：**

| 完成方式 | ①素材量 | ②原形品质/稳定 | ③叙事条目 | 主要玩家类型 |
|---|---|---|---|---|
| 逃离 | 高 | 无 | 低 | 搜刮党、资源优先型 |
| 击杀 | 中 | 中品质/低稳定 | 中 | Build 党、战斗倾向 |
| 收容 | 低 | 高品质/高稳定 | 高 | Lore 收集党、养成深度型 |

### 逃离目标

完成条件：

- 找到出口或逃生路径。
- 解开最低限度路径谜题。
- 玩家活着离开副本。

奖励倾向：见三维结构①高③低。

后果：

- 怪物仍存在。
- 逃离无法获得原形，也无法获得原形残片（仅在完全逃离、未深入时）。
- 后续可能出现该副本变体或追踪事件。

### 击杀目标

完成条件：

- 找到怪物弱点（击杀线索）。
- 获得或制作击杀道具。
- 在正确时机使用弱点。
- 活着完成处置。

奖励倾向：见三维结构①中②战斗向③中。

后果：

- 怪物被破坏，部分真相可能丢失。
- 原形稳定度低，养成风险较高。
- 可能提升玩家战斗路线倾向。
- 击杀不会永久关闭剧情线，但会改变玩家获得信息和原形品质的方式。

### 收容目标

完成条件：

- 收集完整规则线索（收容线索链）。
- 找齐仪式物。
- 按正确顺序执行仪式。
- 避免触发禁忌或在触发后完成补救。

奖励倾向：见三维结构①低②完整③高。

后果：

- 副本异常被稳定封存或转移。
- 基地研究能力提升。
- 可能开启更高阶异常的关注。

### 结算维度

- 完成方式：收容成功、击杀、逃离、错误收容。
- 生存状态：伤势、污染、理智。
- 线索完整度（分层：逃离/击杀/收容各百分比）。
- 资源带回量（维度①）。
- 原形品质和稳定度（维度②）。
- 叙事条目数量（维度③）。
- 副本后续状态（怪物是否还在、副本变体是否触发）。

### 数值显示

结算结果显示明确数值：

- 带回资源数量（分类显示：生存/解谜/养成/情报）。
- 损失资源数量（死亡带入资源损失 + 错误收容仓库损失）。
- 原形品质、稳定度和路线倾向。
- 怪物档案完成度（三维：感知规则/弱点/完整真相各多少%）。
- 叙事条目新增数量。
- 污染变化（玩家污染值 + 基地污染度变化量）。

### 错误收容

错误收容是玩家以错误流程执行收容仪式、触发错误规则或遗漏关键条件后的结果。

错误类型与惩罚分级：

| 错误类型 | 惩罚等级 | 仓库损失 | 是否触发基地入侵 |
|---|---|---|---|
| 仪式残缺（缺少 1–2 条件） | 轻度 | min(仓库总量×5%, 3 单位) | 10% 概率 |
| 规则违反（触发禁忌） | 中度 | min(仓库总量×10%, 8 单位) | 30% 概率 |
| 严重错误（完全错误顺序） | 重度 | min(仓库总量×15%, 15 单位) | 60% 概率 |

使用绝对值上限是为了避免早期仓库空时惩罚失效，以及后期仓库满时惩罚过重。

## 关键流程

```
副本结束触发结算：

判断触发来源（玩家出口/击杀成功/收容成功/错误收容/死亡无法完成）
  ↓
[错误收容] → 计算错误类型 → 查惩罚分级表 → 触发仓库扣除 + 入侵判定
[击杀/收容/逃离] → 计算三维奖励：
  ① 统计背包内物资种类和数量
  ② 从 03 模块获取原形品质和稳定度
  ③ 从 05 模块获取叙事条目数量
  ↓
生成结算数据 → 更新基地仓库 + 原形列表 + 档案
  ↓
显示结算界面（三维奖励 + 状态变化 + 污染变化 + 档案进度）
  ↓
检查后续解锁条件 → 输出给 12-progression
```

## 与其他模块关系

- **线索解谜模块** → 获取目标完成条件
- **怪物规则模块** → 获取击杀和收容结果
- **搜刮资源模块** → 获取带回物资
- **原形养成模块** ← 输出原形品质
- **基地管理模块** ← 输出三维奖励和错误收容惩罚
- **长期成长模块** ← 输出副本评价、解锁和世界线变化

## 第一阶段原型范围

第一阶段实现一个副本的四种结算：

- 逃离成功（三维：①高②无③低）。
- 击杀成功（三维：①中②战斗向/低稳③中）。
- 收容成功（三维：①低②高品质/高稳③高）。
- 错误收容（惩罚分级表 + 基地入侵判定）。

每种结算至少影响：带回素材数量、是否获得原形、原形品质和稳定度、怪物档案完整度、叙事条目新增。

## 验收标准（Acceptance Criteria）

- [ ] 玩家能清楚知道自己以哪种方式完成副本（结算界面明确显示）。
- [ ] 三种目标的奖励差异在三维上均有体现，不只是"素材多少"。
- [ ] 收容奖励明显更好（②和③维度），但过程更危险（需要更多暴露时间）。
- [ ] 击杀不是收容的简单下位替代，而是另一种策略（搜刮党 + 战斗向原形）。
- [ ] 错误收容惩罚明显，损失数值在结算界面可见。
- [ ] 结算界面污染变化和档案完成度都有显示。

## KPI 指标

| 指标名 | 目标值 | 测量方式 |
|---|---|---|
| 三种目标路径都被玩家主动尝试的比率 | ≥70% 玩家在 5 局内尝试过至少 2 种 | 数据统计 |
| 搜刮党首选逃离的比率 | ≥50% | 数据统计 |
| Lore 玩家首选收容的比率 | ≥40% | 数据统计 |
| 错误收容惩罚后玩家下次改变策略的比率 | ≥60% | 数据统计 |

## 风险与依赖

- **依赖：** 05-clues-puzzles（目标完成条件）、03-monster-anomaly-rules（处置结果）、07-looting-resources（资源带回量）、10-base-management（仓库状态）。
- **风险：** R-03（三目标塌缩），三维奖励结构是核心缓解手段；R-05（死亡惩罚定义不明），与 07 模块联动定义，见 07 模块。

## 后续扩展方向

- 多目标混合，例如先击杀分身再收容本体。
- 隐藏结算，例如污染逃离、错误收容、交易异常。
- 长期阵营倾向：逃避者、猎杀者、收容者、利用者。
- 三维奖励的历史统计，用于结局变量计算。

## 已确认设计决策

- 逃离无法获得原形残片（纯逃离时）。
- 击杀不会永久关闭剧情线，但会改变信息和原形品质。
- 结算收束为四种：收容成功、击杀、逃离、错误收容。
- 三维奖励结构（素材量/原形品质/叙事条目）取代单一素材奖励，缓解 R-03。
- 错误收容惩罚采用分级制度（轻度/中度/重度），使用绝对值上限避免极端情况。
- 结算结果显示明确数值。
