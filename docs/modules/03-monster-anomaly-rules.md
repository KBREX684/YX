# 怪物与异常规则模块

版本：v0.3.2
关联总设定版本：v0.8.1

创建日期：2026-05-14
最后更新：2026-05-14

## 模块定位

怪物与异常规则模块负责定义每个怪物的行为逻辑、感知方式、弱点、收容条件和原形产出。它是游戏恐怖感和规则推理的核心。

本游戏中的怪物不应只是追逐玩家的敌人，而应是一套可观察、可学习、可误判、可反制的异常规则。

## 关联 Design Pillars

- **Pillar 1（看不见但可学习的怪物）：** 本模块是 Pillar 1 的直接载体。所有怪物的感知方式、行动逻辑和规则都必须可被玩家推理，弱点和收容条件必须能被副本内线索覆盖。
- **Pillar 2（任何变强都同时意味着变危险）：** 反学习机制是 Pillar 2 在怪物侧的体现：玩家对规则的理解越深（多次相遇），怪物变体规则越复杂，挑战始终存在。

## 设计目标

- 每个怪物拥有清晰的恐惧主题。
- 怪物不可见或不完全可见，但玩家能通过反馈判断状态。
- 怪物规则可以被线索、环境和行为验证。
- 击杀和收容使用同一套怪物规则的不同理解深度。
- 怪物产出的原形与它的恐惧主题和规则有关。
- 每只怪物可通过规则原语组合快速量产，避免从零设计。

## 核心功能

### 怪物档案字段

每个怪物至少需要定义（完整模板见 `docs/monsters/_monster-template.md`）：

- 名称或代号。
- 恐惧主题。
- 外在现象。
- 感知方式（用规则原语描述）。
- 行动阶段（AI 状态机：潜伏/试探/搜索/追猎/处置）。
- 接近反馈（声音/光源/环境/画面）。
- 攻击条件。
- 失败惩罚。
- 弱点条件（击杀链）。
- 收容条件（收容链和禁忌）。
- 原形类型和产出差异。
- 可被原形能力影响的规则。
- 规则原语组成表。
- 反学习机制与变体。

### 异常规则原语系统

规则原语是设计和描述怪物规则的最小单元，共五类（详见术语表）：

| 类型 | 描述 | 设计用途 |
|---|---|---|
| 禁止类 | 玩家绝对不能做某事 | 制造紧绷、限制行动 |
| 必须类 | 玩家必须完成某行为 | 制造任务门槛 |
| 条件触发类 | 满足 X 触发 Y | 构建因果逻辑 |
| 镜像反向类 | 直觉安全=实际危险 | 制造学习惊喜和误判 |
| 感知欺骗类 | 信息本身是假的 | 制造不确定性和验证需求 |

每只怪物应由 2–4 个原语组合构成完整规则体系。原语组合的多样性是量产怪物的基础工具。

### 感知方式

怪物可以通过不同方式锁定玩家（对应原语类型）：

- 声音（奔跑、翻找、破门）→ 禁止类或条件触发类。
- 光线（手电照射特定方向/开关灯）→ 禁止类或镜像反向类。
- 视线（玩家回头或直视特定对象）→ 禁止类。
- 情绪（理智过低时更容易被感知）→ 条件触发类。
- 规则（违反副本特定规则后被标记）→ 条件触发类。
- 物品（携带特定仪式物或线索吸引它）→ 条件触发类。
- 假声源（模拟声音引导玩家主动触发禁止类规则）→ 感知欺骗类。

### 行动阶段（AI 状态机）

每个怪物的行为拆分为五阶段：

1. **潜伏（Dormant）：** 只出现轻微反馈，玩家主要探索。
2. **试探（Probing）：** 制造环境变化，引导玩家理解规则。
3. **搜索（Searching）：** 开始根据声音、光源或规则追踪玩家。
4. **追猎（Hunting）：** 进入高压状态，玩家必须躲藏、逃脱或使用道具。
5. **处置（Disposal）：** 玩家尝试击杀、收容或逃离；或玩家被抓触发惩罚。

每个阶段的切换条件、玩家可感知信号和逃脱条件必须在 Monster Bible 中明确定义。

### 反学习机制

玩家在同一副本多次相遇同一怪物后，怪物启用变体规则，维持学习曲线和多周目恐怖感（对应 R-01 体验曲线塌陷的缓解，以及 R-13 掌控感保护）。

**触发规则：**

- 第 3 次相遇后：启用变体规则 A（轻微调整，如感知范围扩大或感知类型增加）。
- 第 5 次相遇后：启用变体规则 B（中等调整，如击杀窗口缩短）。
- 第 8 次相遇后：启用变体规则 C（强调整，如新增一类感知方式）。

**告知原则（重要）：**

变体存在必须在怪物档案中预先标注"该异常存在变种"。玩家通过怪物研究可以预期变体方向。**禁止在无任何档案预警的情况下激活变体规则。** 变体是让玩家期待学习，而不是被突然改规则所惩罚（Pillar 1 要求）。

### 现形频率契约

怪物现形是感知体验的关键事件，需要符合以下规约（对应 R-02 缓解）：

| 节点 | 规则 |
|---|---|
| 首次进入副本 | 强制现形 1 次（弱光、2–3 秒），让玩家获得基础外观信息 |
| 后续副本 | 由理智值 + 异常事件累积控制，非随机 |
| 每副本现形上限 | 每个怪物设置"现形预算"，防止频率过高失去神秘感 |
| 现形冷却 | 每次现形后有最短冷却时间（具体值在 Monster Bible 中定义） |

### 弱点与收容

- 弱点用于击杀，通常只需要理解怪物的部分规则（击杀链）。
- 收容需要完整规则链，包括来源、禁忌、仪式物和执行顺序（收容链）。
- 击杀可能破坏情报或降低原形稳定度。
- 收容风险更高，但奖励更完整。
- 错误收容条件必须在 Monster Bible 中明确列出。

## 关键流程

```
怪物每帧判断流程：

读取玩家状态（位置/噪音/光源/理智/携带物）
  ↓
当前 AI 阶段判断
  ├─ 潜伏 → 检查切换条件 → [条件满足] 进入试探
  ├─ 试探 → 执行环境变化 → 检查感知规则触发
  ├─ 搜索 → 沿路径追踪上次已知位置 → 检查追猎条件
  ├─ 追猎 → 高速追踪 → 检查处置条件
  └─ 处置 → 触发抓取 OR 处置结果（击杀/收容成功）

感知规则检查（独立于阶段，任意阶段可触发）：
  对每条规则原语（P1–Pn）执行条件判断
  → 触发则对当前阶段施加升级信号
  → 感知欺骗类规则在此处生成假声源事件
```

## 第一阶段原型范围

第一阶段只设计一个怪物（大只，见 `docs/monsters/001-da-zhi.md`）：

- 主要不可见，通过声音和手电异常接近玩家。
- 规则原语：2 条禁止类 + 1 条条件触发类 + 1 条镜像反向类 + 1 条感知欺骗类。
- 首次副本强制现形一次（弱光，2–3 秒）。
- 弱点：广播室断电 + 噪音诱饵引导（击杀链 3 步）。
- 收容：名单确认 + 声音封印 + 实体锚定（收容链 3 步，含 4 条禁忌）。
- 变体规则：第 3 / 5 / 8 次相遇后依次激活。

## 验收标准（Acceptance Criteria）

- [ ] 玩家能在不直接看见怪物的情况下判断危险阶段（潜伏/搜索/追猎可区分）。
- [ ] 玩家能通过线索理解至少一个怪物规则原语。
- [ ] 怪物的首次现形在首次进入副本时必然触发。
- [ ] 玩家能选择逃离、击杀或收容，三条路径都可到达处置节点。
- [ ] 击杀和收容产出的原形品质/稳定度明显不同。
- [ ] 第三次相遇后变体规则 A 激活，档案中有预先标注。

## KPI 指标

| 指标名 | 目标值 | 测量方式 |
|---|---|---|
| 首次副本死亡次数 | 1–3 次（允许学习） | 录像回放 |
| 玩家能说出至少一条怪物规则的比率 | ≥60%（首局后） | 测试后访谈 |
| 收容成功率（已找齐线索的玩家） | ≥50% | 数据统计 |
| 误判率（玩家以为安全但被抓） | 20%–40%（可学习范围） | 录像回放 |

## 风险与依赖

- **依赖：** 04-horror-perception（恐怖反馈表现）、05-clues-puzzles（规则线索分布）、02-dungeon-generation（活动区域）。
- **风险：** R-01（体验曲线塌陷），反学习机制是缓解手段；R-02（现形频率），已通过频率契约缓解；R-06（大只缺乏 spec），已通过 Monster Bible 解决；R-13（变体破坏掌控感），已通过档案预告原则缓解。

## 后续扩展方向

- 多怪物规则组合（两只怪同一副本，规则相互干扰）。
- 同一怪物的高阶变体（变体 D 及以上）。
- 原形之间与怪物规则的克制关系量产化。
- 错误收容导致怪物污染原形或进入基地。

## 已确认设计决策

- 怪物在特定条件下现形，首次副本强制一次（弱光，2–3 秒）。
- 玩家被抓后触发恐怖动画，并在基地复活。
- 怪物可以跨房间追踪，但必须有固定行为逻辑（AI 状态机）。
- 异常规则原语分五类，每个怪物由 2–4 个原语组合。
- 反学习变体在第 3/5/8 次相遇后激活，需在档案中预先标注。
- 第一只怪物名为"大只"，主题为破败校园，见 docs/monsters/001-da-zhi.md。
