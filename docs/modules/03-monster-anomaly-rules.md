# 怪物与异常规则模块

版本：v0.1.0

创建日期：2026-05-14

## 模块定位

怪物与异常规则模块负责定义每个怪物的行为逻辑、感知方式、弱点、收容条件和原形产出。它是游戏恐怖感和规则推理的核心。

本游戏中的怪物不应只是追逐玩家的敌人，而应是一套可观察、可学习、可误判、可反制的异常规则。

## 设计目标

- 每个怪物拥有清晰的恐惧主题。
- 怪物不可见或不完全可见，但玩家能通过反馈判断状态。
- 怪物规则可以被线索、环境和行为验证。
- 击杀和收容使用同一套怪物规则的不同理解深度。
- 怪物产出的原形与它的恐惧主题和规则有关。

## 核心功能

### 怪物档案字段

每个怪物至少需要定义：

- 名称或代号。
- 恐惧主题。
- 外在现象。
- 感知方式。
- 行动逻辑。
- 接近反馈。
- 攻击条件。
- 失败惩罚。
- 弱点条件。
- 收容条件。
- 原形类型。
- 可被原形能力影响的规则。

### 感知方式

怪物可以通过不同方式锁定玩家：

- 声音：奔跑、开门、翻找物品会吸引它。
- 光线：手电照射、开灯或关灯会触发它。
- 视线：玩家看见某物、回头或直视错误对象会触发。
- 情绪：精神压力过高时更容易被感知。
- 规则：违反副本规则后被标记。
- 物品：携带某个线索或仪式物会吸引它。

### 行动阶段

建议将怪物行为拆成阶段：

1. 潜伏：只出现轻微反馈，玩家主要探索。
2. 试探：制造环境变化，引导玩家理解规则。
3. 搜索：开始根据声音、光源或规则追踪玩家。
4. 追猎：进入高压状态，玩家必须躲藏、逃脱或使用道具。
5. 处置：玩家尝试击杀、收容或逃离。

### 弱点与收容

- 弱点用于击杀，通常只需要理解怪物的一部分规则。
- 收容需要完整规则链，包括来源、禁忌、仪式物和执行顺序。
- 击杀可能破坏情报或降低原形稳定度。
- 收容风险更高，但奖励更完整。

## 输入与输出

输入：

- 玩家位置、噪音、光源、精神状态和携带物。
- 地图区域和异常事件。
- 线索解谜模块提供的规则进度。
- 原形助战模块提供的能力干预。

输出：

- 怪物状态和威胁等级。
- 恐怖反馈触发条件。
- 攻击、追击、离开、搜索等行为事件。
- 击杀或收容结果。
- 原形产出信息。

## 与其他模块关系

- 与恐怖感知模块共同表现怪物接近和规则触发。
- 与地图模块共享活动区域、触发区和禁区。
- 与线索解谜模块共享规则揭示和验证。
- 与目标结算模块共享击杀和收容结果。
- 与原形养成模块共享原形类型、品质和稳定度。

## 第一阶段原型范围

第一阶段只设计一个怪物：

- 主要不可见。
- 通过声音和手电异常接近玩家。
- 有一个明确弱点。
- 有一套三步收容仪式。
- 被击杀和被收容时产出不同品质原形。

示例方向：走廊里模仿玩家脚步的异常。玩家奔跑时它接近，玩家回头时手电闪烁，玩家在错误教室读出名字会被标记。

## 第一阶段验收标准

- 玩家能在不直接看见怪物的情况下判断危险。
- 玩家能通过线索理解至少一个怪物规则。
- 玩家能选择逃离、击杀或收容。
- 击杀和收容带来的原形结果不同。

## 后续扩展方向

- 多怪物规则组合。
- 同一怪物的高阶变体。
- 怪物对玩家习惯的学习。
- 原形之间与怪物规则的克制关系。
- 收容失败导致怪物进入基地或污染原形。

## 待确认问题

- 怪物是否永远不可见，还是只在特定条件下显形。
- 玩家被抓后是即死、受伤、污染，还是进入特殊空间。
- 怪物是否能跨房间追踪，还是以区域威胁为主。
- 第一只怪物的主题和名称。
