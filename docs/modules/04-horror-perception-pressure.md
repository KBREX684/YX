# 恐怖感知与压力模块

版本：v0.3.0
关联总设定版本：v0.6.0
状态：现形频率契约与理智阈值系统确认
创建日期：2026-05-14
最后更新：2026-05-14

## 模块定位

恐怖感知与压力模块负责把怪物和异常规则转化为玩家能感受到的声音、光线、环境、生理和画面反馈。它承担"怪物看不见，但危险正在逼近"的表达任务。

这个模块的重点是可学习的恐怖，而不是随机惊吓。

## 关联 Design Pillars

- **Pillar 1（看不见但可学习的怪物）：** 本模块是 Pillar 1 的表现层。每一类反馈都必须对应具体的怪物规则或异常事件，让玩家能从混乱信号中学习，而不是只被随机惊吓。
- **Pillar 2（任何变强都同时意味着变危险）：** 理智系统是 Pillar 2 在感知层的体现：玩家携带的原形越强，污染越高，感知干扰越多，副本本身的读取难度会随之上升。

## 设计目标

- 让玩家通过间接信号判断怪物状态。
- 让压力逐步升级，而不是突然从安全跳到死亡。
- 允许误判，但误判应来自玩家理解不足，而不是系统随机。
- 让声音、手电、环境、心跳和画面异常都能对应具体规则。
- 支持不同怪物有不同的恐怖反馈语言。

## 核心功能

### 声音反馈

- 远近变化：脚步、敲击、拖拽声随距离改变。
- 方向误导：怪物可以模拟错误声源（感知欺骗类规则原语的表现层）。
- 材质差异：木地板、瓷砖、金属管道传来的声音不同。
- 低语信息：可能是真线索，也可能是诱导。
- 静默压力：危险极高时声音突然消失。

### 光源反馈

- 手电轻微闪烁：怪物接近或规则轻度触发。
- 手电剧烈闪烁：追猎状态或高污染区域。
- 照明范围缩小：理智下降或异常压制。
- 短暂显影：照出痕迹、影子、脚印或仪式符号。
- 光源反噬：错误使用光源可能吸引怪物（部分怪物的镜像反向类规则）。

### 环境反馈

- 物品错位。
- 门自动开合。
- 墙面浮现文字或痕迹。
- 房间温度、雾气、尘埃变化。
- 地图结构改变（配合副本地图事件）。
- 资源或线索被移动。

### 理智系统

理智（Sanity）是玩家在副本中的精神状态参数，影响画面、声音和解谜信息的可靠性（详见术语表）。

**理智输入事件（消耗）：**

| 事件 | 消耗量 |
|---|---|
| 看到怪物现形 | 中（-15） |
| 长时间处于黑暗中 | 小（-3/分钟） |
| 怪物进入追猎阶段 | 中（-10） |
| 处于高污染区域 | 小（-5/分钟） |
| 携带恐怖型原形且稳定度低 | 小（-3/分钟） |
| 目睹死亡动画（被处置一次） | 大（-25，复活后） |

**理智回复方式：**

| 来源 | 回复量 |
|---|---|
| 基地休整（副本外） | 完全恢复 |
| 镇定剂（副本内消耗品） | 中（+20） |
| 拟人化原形稳定互动 | 小（+10，每次副本限 2 次） |
| 进入相对安全区（无追踪状态）持续 60 秒 | 小（+5） |

**理智阈值与干扰表（5 段）：**

| 阈值段 | 理智值范围 | 干扰效果 |
|---|---|---|
| 稳定 | 80–100 | 无干扰 |
| 轻度压力 | 60–79 | 手电轻微闪烁加剧，远处偶尔有异声 |
| 中度压力 | 40–59 | 线索文本偶尔乱码或顺序错误；操作轻微抖动 |
| 重度压力 | 20–39 | 方向性声音失真（左右混淆）；仪式物品提示可能显示错误位置 |
| 崩溃边缘 | 0–19 | 恐怖型原形无效化；拟人型原形可能失控判定触发；画面严重扭曲；线索几乎不可读 |

**理智与原形耦合规则：**

- 理智低于 20% 时：恐怖型原形的所有主动能力无效化（见 09-origin-companion-support）。
- 理智低于 20% 时：拟人型原形的失控判定概率提升（见 09-origin-companion-support）。
- 理智不影响工具型原形的基本功能，但高污染值 + 低理智叠加时，工具型原形可能出现误报。

### 画面反馈

- 噪点。
- 色偏。
- 拖影。
- 画面撕裂或短暂黑屏。
- 镜头节奏干扰。
- 低理智下出现不可靠感知（幻影、错误轮廓）。

### 表现原则

- 不使用 UI 明示危险、理智或压力数值。
- 玩家完全依靠声音、心跳、画面、光源和环境表现判断状态。
- 心跳主要作为音效反馈使用，不依赖界面提示。
- 每类反馈必须对应具体规则或事件，不做纯装饰性随机。

## 关键流程

```
每帧恐怖反馈更新：

输入：怪物威胁等级 + AI 阶段 + 玩家状态
  ↓
计算当前应激活的声音/光/环境/画面层级
  ↓
应用理智消耗事件（根据当前触发事件查表）
  ↓
更新理智阈值段 → 输出线索可靠性系数
  ↓
触发音频事件 + 光源状态 + 环境变化 + 画面后处理
  ↓
检查理智 < 20% → 触发原形互锁信号（恐怖型无效化 / 拟人型失控检测）
```

## 与其他模块关系

- **怪物规则模块** → 接收威胁状态，驱动反馈强度
- **玩家控制模块** → 接收体力、污染和光源信息；输出理智给玩家参数系统
- **线索解谜模块** ← 输出线索可靠性系数，低理智时影响线索读取
- **原形助战模块** ← 输出理智值，驱动原形互锁规则（恐怖型无效化、拟人型失控）
- **目标结算模块** → 接收污染等结算数据

## 第一阶段原型范围

第一阶段实现五类反馈，每类对应明确的怪物状态：

- 声音远近（怪物距离→音量和方向变化）。
- 手电闪烁（威胁等级→闪烁频率）。
- 心跳加速（AI 阶段→心跳节奏）。
- 物品错位（试探阶段触发）。
- 画面轻微噪点或色偏（理智降低时触发）。
- 低理智（< 60%）对线索文本造成轻度干扰。

理智阈值第一阶段只实现 3 段（稳定/中度/崩溃），完整 5 段在第二阶段引入。

## 验收标准（Acceptance Criteria）

- [ ] 玩家能根据反馈判断危险大致阶段（潜伏/搜索/追猎可区分）。
- [ ] 玩家能通过反馈学习"什么行为吸引怪物"。
- [ ] 同一反馈在不同强度下代表不同风险级别。
- [ ] 反馈不会遮挡关键操作和文本阅读（高压下仍可完成仪式交互）。
- [ ] 理智低于中度压力阈值时，至少一类线索出现轻度干扰。
- [ ] 理智低于崩溃阈值时，恐怖型原形无效化信号正确触发。

## KPI 指标

| 指标名 | 目标值 | 测量方式 |
|---|---|---|
| 玩家能通过反馈预判追猎开始的比率 | ≥50% | 录像回放（玩家是否提前进入躲藏点） |
| 因反馈混乱导致操作失误的比率 | ≤20%（在可接受学习范围内） | 录像回放 |
| 理智崩溃导致的不公平感报告 | ≤10% 测试者 | 测试后访谈 |

## 风险与依赖

- **依赖：** 03-monster-anomaly-rules（威胁等级和 AI 阶段）、01-player-control（体力和污染）、09-origin-companion（原形类型和稳定度）。
- **风险：** R-02（现形频率失控），已通过与 03 模块协同定义的频率契约缓解；R-08（理智系统三方耦合），已通过本模块集中定义阈值表、输入表和原形互锁规则缓解。

## 后续扩展方向

- 完整理智 5 段阈值系统（第二阶段）。
- 每个怪物拥有专属反馈组合（差异化音效和视觉语言）。
- 假线索和真实线索的感知区别（结合理智系统）。
- 原形携带改变恐怖反馈（恐怖型原形提升怪物接近的反馈强度）。
- 不同玩家状态下感知信息失真方式多样化。

## 已确认设计决策

- 不使用 UI 明示状态，完全依赖表现反馈。
- 心跳作为音效反馈。
- 理智系统取代幻觉概念，理智低时影响解谜。
- 理智阈值分 5 段，阈值 20% 以下触发原形互锁规则。
- 理智输入事件表和回复方式表已在本文档中定义。
