# 恐怖感知与压力模块

版本：v0.2.0
关联总设定版本：v0.4.0
状态：初版细化
创建日期：2026-05-14
最后更新：2026-05-14

## 模块定位

恐怖感知与压力模块负责把怪物和异常规则转化为玩家能感受到的声音、光线、环境、生理和画面反馈。它承担“怪物看不见，但危险正在逼近”的表达任务。

这个模块的重点是可学习的恐怖，而不是随机惊吓。

## 设计目标

- 让玩家通过间接信号判断怪物状态。
- 让压力逐步升级，而不是突然从安全跳到死亡。
- 允许误判，但误判应来自玩家理解不足，而不是系统随机。
- 让声音、手电、环境、心跳和画面异常都能对应具体规则。
- 支持不同怪物有不同的恐怖反馈语言。

## 核心功能

### 声音反馈

- 远近变化：脚步、敲击、拖拽声随距离改变。
- 方向误导：怪物可以模拟错误声源。
- 材质差异：木地板、瓷砖、金属管道传来的声音不同。
- 低语信息：可能是真线索，也可能是诱导。
- 静默压力：危险极高时声音突然消失。

### 光源反馈

- 手电轻微闪烁：怪物接近或规则轻度触发。
- 手电剧烈闪烁：追猎状态或高污染区域。
- 照明范围缩小：理智下降或异常压制。
- 短暂显影：照出痕迹、影子、脚印或仪式符号。
- 光源反噬：错误使用光源可能吸引怪物。

### 环境反馈

- 物品错位。
- 门自动开合。
- 墙面浮现文字或痕迹。
- 房间温度、雾气、尘埃变化。
- 地图结构改变。
- 资源或线索被移动。

### 理智与生理反馈

- 心跳速度表示危险压力。
- 呼吸声表示体力和恐惧状态。
- 视野收缩表示理智下降。
- 操作轻微抖动表示高压状态。
- 理智过低时会影响解谜，例如线索阅读变形、符号顺序误判、声音提示失真。
- 污染值过高时会加速理智消耗或强化异常表现。

### 画面反馈

- 噪点。
- 色偏。
- 拖影。
- 画面撕裂或短暂黑屏。
- 镜头节奏干扰。
- 低理智下出现不可靠感知。

### 表现原则

- 不使用 UI 明示危险、理智或压力数值。
- 玩家完全依靠声音、心跳、画面、光源和环境表现判断状态。
- 心跳主要作为音效反馈使用，不依赖界面提示。

## 输入与输出

输入：

- 怪物状态、距离、感知阶段和规则触发。
- 玩家理智、污染值、体力和受伤状态。
- 地图异常区域信息。
- 原形能力副作用。

输出：

- 音频事件。
- 光源异常。
- 环境变化。
- 画面后处理状态。
- 玩家理智和压力参数变化。

## 与其他模块关系

- 从怪物规则模块接收威胁状态。
- 从玩家模块接收体力、精神、污染和光源使用信息。
- 向线索解谜模块提供可被观察的规则表现。
- 向原形助战模块提供能力副作用表现。
- 向目标结算模块提供压力、污染等结算数据。

## 第一阶段原型范围

第一阶段只做五类反馈：

- 声音远近。
- 手电闪烁。
- 心跳加速。
- 物品错位。
- 画面轻微噪点或色偏。
- 低理智对解谜信息造成轻度干扰。

这些反馈必须能对应怪物状态，而不是纯装饰。

## 第一阶段验收标准

- 玩家能根据反馈判断危险大致阶段。
- 玩家能通过反馈学习“什么行为吸引怪物”。
- 同一反馈在不同强度下代表不同风险。
- 反馈不会遮挡关键操作和文本阅读。

## 后续扩展方向

- 每个怪物拥有专属反馈组合。
- 理智异常系统。
- 假线索和真实线索的感知区别。
- 原形携带改变恐怖反馈。
- 不同玩家状态下感知信息失真。

## 已确认设计决策

- 不使用 UI 明示状态，完全依赖表现。
- 心跳作为音效反馈。
- 幻觉概念调整为理智系统，理智低时会影响解谜。
- 新手期仍需要通过场景和事件教会玩家理解反馈语言。
