## 用途

集中管理所有需要统一口径的设计问题、工程选型问题和后续阶段问题。过去散落在各模块底部的“待设计问题”统一迁移到本文件。

截至 v1.5.0，Q-01 ~ Q-20 均已定案；当前没有需要用户继续确认的阻塞项。剧情、关卡内容、Demo 文案、怪物弱点细节、结局变量细节和美术制作细节，由项目默认角色“游戏专业总设计师 + 总工程师”在不偏离总设定基底和路线的前提下自行定案并回填。

**状态说明：**
- ✅ 已定案：用户或设计负责人已确认，相关文档必须与决策保持一致
- 🟡 后续阶段细化：不阻塞当前阶段，已有默认方案，后续实现时按实测调参
- 🔴 阻塞：未来新增问题若影响当前施工，必须先定案再推进

更新规则：决策做出后标注“已定案”，写入决策内容、定案日期和关联文档更新。

## 已定案问题

### Q-01 玩家具体身份来源

- **状态：** ✅ 已定案
- **来源模块：** 11-narrative-worldbuilding
- **影响范围：** 叙事与世界观模块、电台 NPC 对话、第一副本进入理由
- **决策内容：** 玩家采用“异常事件幸存者 + 旧机构候选执行者”的混合身份。玩家在某次异常事件中唯一存活，旧收容所的半自动协议因此把玩家识别为最后可用的执行者候选。
- **第一阶段用法：** Demo 中玩家不具备完整收容知识；电台第一通话只说明“你是唯一活下来的人，基地把你登记为执行者”，让玩家与角色一起学习规则。
- **设计理由：** 保留陌生感和恐怖感，同时让“遗留收容所”闭环成立。
- **定案日期：** 2026-05-14

### Q-02 电台 NPC 的真实立场与可信度

- **状态：** ✅ 已定案
- **来源模块：** 10-base-management-research / 11-narrative-worldbuilding
- **影响范围：** 电台对话内容、电台关系结局变量、旧机构真相叙事结构
- **决策内容：** 第一阶段电台是旧机构遗留的通讯人格/录音系统，立场偏中立，不主动撒谎，但有信息盲区和协议残缺。它提供可用指引，也会因档案缺失、旧规则失效或协议冲突而给出不完整信息。
- **可疑信号用法：** 与档案矛盾、预测失败、关键沉默三类信号作为后续裂缝伏笔；Demo 中不直接判定电台背叛。
- **定案日期：** 2026-05-14

### Q-03 旧收容机构的核心罪行与最终真相

- **状态：** ✅ 已定案
- **来源模块：** 11-narrative-worldbuilding / 12-progression-difficulty-longterm-growth
- **影响范围：** 主叙事后期、真相闭合结局族群、旧机构真相完成度变量
- **决策内容：** 旧机构的核心罪行是同时把人类和异常都当作“收容材料”使用：人类被协议、记忆、身体或身份绑定到副本稳定结构中；异常被塑形成工具、守卫、能源或原形来源。
- **第一阶段用法：** 破败校园只投放 1~2 条残页或电台矛盾，暗示“旧收容所曾在此做过一次失败收容”，不在 Demo 中完整揭露罪行。
- **定案日期：** 2026-05-14

### Q-04 结局族群进入阈值

- **状态：** ✅ 已定案
- **来源模块：** 12-progression-difficulty-longterm-growth
- **影响范围：** 结局触发条件、长期倾向变量权重
- **决策内容：** 采用固定阈值方案。主轴任一方向占比 ≥ 70% 时进入对应极；45%~55% 视为模糊带，结局表述保留摇摆和代价；55%~70% 视为偏向但未极化。
- **设计理由：** 第一版计算清晰、可调参、易解释；第三阶段如需要更细腻再演进为加权投票。
- **定案日期：** 2026-05-14

### Q-05 电台关系变量的计算方式

- **状态：** ✅ 已定案
- **来源模块：** 10-base-management-research / 12-progression-difficulty-longterm-growth
- **影响范围：** 修饰位变量、结局族群内部变体
- **决策内容：** 电台关系使用 `radio_relation_score`，范围 -100 ~ +100。负值代表怀疑/揭露倾向，正值代表信任/继承倾向。
- **基础计分：**
  - 按电台建议行动且未交叉验证：+5
  - 发现电台与档案矛盾并记录：-8
  - 电台预测失败后仍继续依赖：+3
  - 电台预测失败后选择自行验证：-10
  - 关键时刻主动违背电台建议：-12
  - 电台修正自身错误并被玩家接受：+3
- **第一阶段用法：** 只后台记录，不向玩家显示。
- **定案日期：** 2026-05-14

### Q-06 破败校园与旧机构最终真相的关系

- **状态：** ✅ 已定案
- **来源模块：** 12-progression-difficulty-longterm-growth / 11-narrative-worldbuilding
- **影响范围：** 真相闭合结局族群、首个副本叙事深度
- **决策内容：** 破败校园不是旧机构核心罪行的最初现场，而是一个独立异常事件。旧机构后来把这里作为观察、训练和失败收容记录场，留下了仪式残页、广播协议和一次失败收容痕迹。
- **第一阶段用法：** Demo 聚焦玩法验证；只揭示“旧收容所曾来过这里”，不把世界观真相一次性讲完。
- **定案日期：** 2026-05-14

### Q-07 原形数量上限与收容室容量

- **状态：** ✅ 已定案
- **来源模块：** 10-base-management-research / 08-origin-acquisition-growth
- **影响范围：** 资源管理深度、长期玩家与新手差距
- **决策内容：** 第一阶段收容室容量为 3 个原形；第二阶段基地基础容量为 6，升级后上限为 8。超过容量后，最低稳定度且最近未携带的原形自动进入休眠；唤醒需要消耗稀有稳定素材。玩家后续可以主动休眠、牺牲或交易特殊资源。
- **设计理由：** 防止长期无限囤积，同时保留“原形不会死亡”的承诺。
- **定案日期：** 2026-05-14

### Q-08 大只击杀弱点的具体设计

- **状态：** ✅ 已定案
- **来源模块：** 03-monster-anomaly-rules / docs/monsters/001-da-zhi.md
- **影响范围：** 第一阶段原型的击杀目标路径
- **决策内容：** 大只的弱点是“校园广播依附”。击杀路径为：获取仓库备用电源钥匙 → 在潜伏/搜索阶段进入广播室关闭总电源 → 大只进入失向状态 → 在仓库入口放置噪音诱饵 → 等大只趋近后触发门锁机关封锁 → 进入击杀结算。
- **结果约束：** 这次“击杀”是规则层面的封锁/处置，不永久关闭剧情线；产出恐怖化倾向、中品质、低稳定度原形，并损坏部分来源记忆。
- **定案日期：** 2026-05-14

### Q-13 2.5D Live 资产规格基准

- **状态：** ✅ 已定案
- **来源模块：** 00-tech-constraints §六
- **影响范围：** 角色、原形、怪物、场景层、视差层、光影层与占位资源规范
- **决策内容：** 不使用像素瓦片规格作为全局美术基准，也不再选择 32×32 / 64×64 瓦片路线。第一阶段采用固定镜头 1080p 内部渲染下的 2.5D Live 分层资产：角色 / 原形 / 怪物使用高分辨率分层透明图、骨骼或网格变形与细微动态；场景使用前景 / 中景 / 背景分层厚涂资源与视差层。
- **执行约束：** `TileMap` 仅可作为灰盒/blockout 或碰撞参考，不作为最终画面方案。最终资源以分层 PNG / PSD 导出、Godot `Sprite2D` / `Parallax2D` / `AnimationPlayer` / `Skeleton2D` / Shader 组合实现。
- **定案日期：** 2026-05-14

### Q-14 美术风格主轴

- **状态：** ✅ 已定案
- **来源模块：** 00-tech-constraints §六
- **影响范围：** 美术外包/采购方向、Shader 体系
- **决策内容：** 美术风格确定为 **厚涂的精美二次元风格**，结合 R16+ 恐怖主题的压抑光影、污损材质和异常变形。主角、拟人原形、怪物现形与关键剧情图以精美二次元厚涂审美为主，强调体积、材质、光影和情绪表现。
- **排除项：** 不使用像素风，不使用低多边形手绘作为主风格。占位素材可以临时使用简化块面或灰盒，但不得反向定义最终美术方向。
- **定案日期：** 2026-05-14

### Q-19 美术制作底基

- **状态：** ✅ 已定案

- **影响范围：** 美术外包、AI 辅助图验收、源文件管理、运行资源导出
- **决策内容：** 第一阶段源文件可使用 Krita / Clip Studio Paint / Photoshop；默认免费底基推荐 Krita。美术工具只负责分层绘制和修图，正式游戏运行资源必须导出为分层 PNG，并由 Godot `.tscn` / `.tres` 管理。

### Q-20 2.5D Live 动画技术路线

- **状态：** ✅ 已定案
- **来源模块：** 00-art-direction / 00-tech-constraints §五 §六
- **影响范围：** 角色动画、原形动画、怪物现形、场景视差、Shader 动效、后续工具评估
- **决策内容：** 第一阶段采用 Godot 原生 2D Live 管线：`AnimationPlayer` + `Skeleton2D` / `Bone2D` / `Polygon2D` + `Sprite2D` / `Parallax2D` + Shader / Light2D。所有动画必须能在 Godot 编辑器内预览和维护。
- **执行约束：** Live2D Cubism 与 Spine 不作为第一阶段底基；垂直切片证明 Godot 原生管线不足后，第二阶段再按插件采纳门槛评估。
- **定案日期：** 2026-05-14
