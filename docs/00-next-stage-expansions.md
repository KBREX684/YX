# 下一阶段扩展清单 Next-Stage Expansions

版本：v1.0.0
关联总设定版本：v0.8.2
创建日期：2026-05-14
最后更新：2026-05-14

## 用途

本文档集中标注当前垂直切片不做、但后续阶段可扩展的内容。它的目的不是削弱项目野心，而是保护第一阶段 Demo 的可完成性：先把核心循环做成可玩、可测、可复盘，再扩大内容量和表现深度。

判定原则：

- 只要某项内容不影响第一阶段验证"看不见但可学习的怪物 + 三目标副本 + 原形养成循环"，就可以延后。
- 延后项必须说明最早进入阶段、延后原因、回归条件。
- 延后项不得在 P1~P6 期间以"顺手做一下"的方式偷偷进入范围。

---

## 已前置到第一阶段的事项

下列事项不再延后，必须在 P1~P3 期间完成最低可用版本：

| 事项 | 最早阶段 | 最低完成口径 |
|---|---|---|
| 输入动作映射（Input Map） | P1-1 | `move_left/right/up/down`、`run`、`crouch`、`flashlight`、`interact`、`hide`、`pause` 全部进入 `project.godot` |
| 微切片验证（Micro Slice） | P1-1 ~ P1-2 | 先做走廊 + 2 个房间 + 1 个躲藏点 + 1 个交互占位物 + 1 次变化事件，再扩展完整破败校园 |
| 资源清单 / 稳定资源 ID | P1 起 | 玩法逻辑引用 manifest key / resource id，不直接依赖裸文件路径 |
| 死亡学习反馈最低版 | P2-1 ~ P3-4 | `RuleResource.learnable_hint` 可在死亡/失败后给出一条可学习提示 |

## 延后到 P7/P8 的事项

| 延后项 | 最早阶段 | 延后原因 | 回归条件 |
|---|---|---|---|
| 死亡复盘演出打磨 | P7 | P3 只需要最低提示；P7 再做节奏、文案和演出强化 | `learnable_hint` 已能稳定显示，并有盲测反馈 |
| 基地"不完全安全"氛围细节 | P7 | 不阻塞核心循环，先保证基地可移动、可养成、可准备 | P5 核心循环跑通后再加异常氛围 |
| 电台首通话与档案页文本扩写 | P7 | 叙事文本会拖慢系统验证，先用克制占位文本 | P6 垂直切片系统验收通过 |
| 完整难度曲线打磨 | P7 | 需要真实盲测数据，不能在系统未闭环前调参 | P5-3 至少一次非开发者盲测完成 |
| CI 与 schema 负例加固 | P8 | 不阻塞本地原型，但影响后续内容生产质量 | P6 导出包稳定后接入 |
| 全部顶层文档收尾与第二阶段立项接口 | P8 | 需要 P0~P7 的实际结果反推 | P7 体验打磨完成 |

## 延后到第二阶段的事项

| 延后项 | 延后原因 | 第二阶段进入条件 |
|---|---|---|
| 第二个副本主题 | 第一阶段只验证破败校园，不扩大关卡制作量 | 破败校园垂直切片完整跑通 |
| 第二只及更多怪物 | 怪物规则量产前必须先验证"大只"模板有效 | 大只 Monster Bible、RuleResource、AI、线索链均可复用 |
| 候选房间池随机化扩大 | 第一阶段可先固定候选房间，降低地图变量 | P1 完整地图可玩，且手工房间模板稳定 |
| 基地设施升级 | 第一阶段基地是循环节点，不做经营深度 | P5 核心循环成立，仓库/收容室/准备区数据稳定 |
| 研究树 | 容易扩大系统量，第一阶段只保留档案与基础研究占位 | 至少两个怪物/副本后再设计研究分支 |
| 多结局展示系统 | 第一阶段保留结局变量，不做完整结局演出 | 第二阶段或第三阶段有足够变量来源后立项 |
| 完整拟人化好感、换装、亲密互动 | 内容量大，且容易冲淡恐怖基调 | 原形三路线基础副作用已验证，R16+ 表现边界复核后进入 |
| 高价值角色最终美术与全套 Live 动画 | 第一阶段先验证可玩性和 Godot 原生 2D Live 管线 | 微切片与完整 VS 均证明角色演出值得投入 |
| Live2D Cubism / Spine 等外部动画运行时 | 授权、插件、导入和维护风险高 | Godot 原生管线被垂直切片证明不足，并通过插件采纳门槛 |

## 延后到第三阶段或更后

| 延后项 | 延后原因 | 回归条件 |
|---|---|---|
| 程序化连通走廊 | 与第一阶段手工可控关卡冲突，容易破坏规则推理公平性 | 第二阶段候选房间池验证稳定后，再评估是否需要 |
| 多怪物同场规则干扰 | 规则复杂度大幅增加，先保证单怪物可学习 | 至少 2 只单独怪物的规则模板成熟 |
| 高阶结局族群完整演出 | 需要足够长期变量和内容积累 | 多副本、多原形、多电台关系事件形成后 |
| 大规模外包资产规范升级 | 需要先证明美术底基和资产拆分方式有效 | 第一阶段美术管线复盘后输出正式外包包 |
