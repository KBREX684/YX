# 垂直切片验收清单 Vertical Slice

版本：v1.0.4
关联总设定版本：v0.8.2
创建日期：2026-05-14
最后更新：2026-05-14

## 用途

定义"第一阶段原型（Vertical Slice）"的完成边界。只有当本清单所有 P0 项被勾选，原型才算验收通过。

垂直切片的目标是验证核心循环是否可行："**在副本中靠声音和规则判断怪物 → 选择目标路径 → 带回原形 → 养成后影响下一次副本**"这个体验是否好玩。

本文档不规定实现方式，只规定"什么算做好了"（Acceptance Criteria）。

视觉与动画实现方式以 [`00-art-direction.md`](00-art-direction.md) 为准：第一阶段使用 Godot 原生 2D Live 管线，不把 Live2D Cubism 或 Spine 作为验收前置。

---

## 微切片门禁（进入完整 VS 前）

微切片是 P1 内部施工门禁，不替代本文件后续 P0 验收项。进入完整破败校园地图和完整美术投入前，必须先完成以下最小可玩验证：

- [ ] `project.godot` 已定义 `move_left`、`move_right`、`move_up`、`move_down`、`run`、`crouch`、`flashlight`、`interact`、`hide`、`pause`，玩家控制脚本不硬编码物理按键。
- [ ] 玩家能在灰盒场景中完成行走、奔跑、蹲伏、开关手电、互动和躲藏。
- [ ] 场景最小范围包含：1 条走廊、2 个房间、1 个躲藏点、1 个可交互占位物（stub）。
- [ ] 至少 1 次地图变化事件可被触发，并且不会阻断逃离占位路径。
- [ ] 至少 1 条噪声事件通过 `EventBus` 发出，并带有稳定 action id / resource id。

死亡或失败后的最低学习反馈不阻塞 P1 微切片，但阻塞 P3 出口：P2-1 必须填入关键失败规则的 `RuleResource.learnable_hint`，P3-4 必须能读取并显示。

---

## 核心循环验证目标

如果以下三个问题的答案都是"是"，原型成立，项目方向可以继续推进：

1. **玩家能在不直接看见怪物的情况下，通过间接反馈判断危险，并做出有意义的决策吗？**
2. **玩家能选择逃离、击杀或收容，并且三种选择带来明显不同的结果吗？**
3. **玩家能把副本成果带回基地，用于养成原形，并在下一次副本中感受到原形的影响吗？**

## P0 验收项（必须全部完成）

### 1. 玩家控制与探索

- [ ] 玩家可以在完整副本中完成：行走、奔跑、蹲伏、开门、拾取物品、阅读线索、进入躲藏点。
- [ ] 奔跑、蹲伏、开灯、关灯会产生不同的怪物判断结果（可通过怪物行为变化验证）。
- [ ] 手电能正常开关，有电量消耗，电量低时有视觉反馈。
- [ ] 玩家死亡后从基地复活，副本状态重置，带入资源损失。

### 2. 恐怖感知与压力

- [ ] 怪物接近时有声音、心跳、手电闪烁三类反馈同时激活。
- [ ] 三类反馈在不同强度下能区分"怪物较远"和"怪物很近"。
- [ ] 玩家能根据反馈判断"我现在是安全还是危险"，而不是只靠碰运气。
- [ ] 第一次进入副本，怪物必须至少现形一次（弱光、2–3 秒），让玩家获得基础外观信息。
- [ ] 理智过低时，至少一类线索信息出现轻度干扰（如文字变形或声音失真）。

### 3. 副本地图（破败校园）

- [ ] 副本包含：入口区、主走廊、4–6 个可进入房间（从候选池抽取）、1 个仪式房、1 个出口区。
- [ ] 地图存在至少一个"变化事件"：走廊变长、门牌错乱或已探索房间出现新物品。
- [ ] 地图变化不会让三条目标路径（逃离/击杀/收容）任何一条变为不可完成。
- [ ] 副本失败后地图重置，但候选房间可以不同（体现重玩变化）。

### 4. 怪物"大只"

- [ ] 大只通常不可见，仅在两种以上特定条件下短暂现形。
- [ ] 大只的接近可以通过间接反馈判断（声音、手电、环境）。
- [ ] 大只遵循明确的 AI 阶段流程（潜伏→试探→搜索→追猎），不做无规则随机传送。
- [ ] 大只有一个可被线索揭示的明确弱点，玩家能用该弱点执行击杀。
- [ ] 大只有一套三步收容仪式，玩家能通过线索推理出仪式内容。
- [ ] 击杀大只产出不稳定原形，收容大只产出稳定原形，两者有明显区别。

### 5. 线索与解谜

- [ ] 副本内有逃离线索 3 条（帮助找出口/开门）。
- [ ] 副本内有击杀线索 3 条（帮助理解弱点和时机）。
- [ ] 副本内有收容线索 5 条（帮助理解仪式物、步骤和禁忌）。
- [ ] 玩家只靠逃离线索可以通关，但没有原形和部分档案。
- [ ] 玩家找到击杀线索后，能在 3 分钟内理解弱点，并知道如何使用。
- [ ] 玩家收集所有收容线索后，能在不查攻略的情况下合理推出仪式流程。
- [ ] 至少一条收容线索可以通过怪物行为反应验证（玩家观察到"怪物对这个有反应"）。

### 6. 副本目标与结算

- [ ] 四种结算均可被触发：逃离成功、击杀成功、收容成功、错误收容。
- [ ] 玩家在结算页能看到：带回资源数量、损失资源数量、原形品质和稳定度、档案完成度、污染变化。
- [ ] 逃离无法获得原形（只获得基础素材和部分情报）。
- [ ] 收容奖励明显优于击杀，击杀奖励明显优于逃离（三维：素材量/原形质量/叙事条目）。
- [ ] 错误收容触发基地物资损失，损失数值在结算页有显示。

### 7. 搜刮与资源

- [ ] 背包存在"带入区"格子上限，玩家不能把整个仓库塞入背包出门。
- [ ] 副本内有 3 类生存物资、3 类解谜物资、3 类养成素材分布。
- [ ] 玩家为了养成素材会考虑是否进入危险区域（有选择压力）。
- [ ] 死亡时：带入资源全失，副本内拾取的资源约 65% 返还。

### 8. 原形养成

- [ ] 玩家能把获得的原形带回基地，向三条路线中的任意一条投喂素材。
- [ ] 0%–60% 阶段，投喂逆向素材会产生路线回拨（代价：稳定度损耗）。
- [ ] 60% 时原形外貌出现明显变化，系统发出"路线即将锁定"提示。
- [ ] 60%–100% 阶段，路线不再改变，只能继续当前路线。
- [ ] 三条路线各有一个第一阶段可带入副本的能力原型（提示/威慑/活体手电）。
- [ ] 每种能力有对应副作用，副作用对副本体验有真实影响。

### 9. 原形携带与助战（简化版）

- [ ] 玩家可以选择是否携带一个原形进入副本。
- [ ] 带入的原形可以在副本内触发对应能力（拟人/恐怖/工具 各一次测试）。
- [ ] 致命伤触发后，原形根据路线产生对应代价（拟人好感降/恐怖污染升/工具损坏）。

### 10. 基地（简化版）

- [ ] 基地是可自由移动的小场景，包含收容室、仓库、准备区和档案入口。
- [ ] 玩家在基地能看到副本带回的素材和当前原形状态。
- [ ] 玩家能在基地选择下一次副本携带的原形。
- [ ] 基地存在污染度显示（即使第一阶段不显示数值，也要有视觉提示）。

### 11. 核心循环完整性

- [ ] 完成一次完整循环：基地准备 → 进入副本 → 探索/躲藏/完成目标 → 结算 → 基地养成 → 再次进入副本。
- [ ] 带着养成后的原形进入副本时，体验与第一次进入明显不同（至少原形能力改变了一次副本场景）。
- [ ] 不同完成方式（逃离/击杀/收容）影响了基地内可用的素材和原形，并因此影响了下一次准备。

## P1 验收项（提升体验，但不阻塞核心验收）

- [ ] 基地有"不完全安全"的视觉/声音提示（异常已被带回的氛围感）。
- [ ] 电台有基础语音或文本引导（第一通话解释玩家身份和基地功能）。
- [ ] 大只的档案页在收容成功后有明显内容增加（奖励信息感）。
- [ ] 副本失败后有一句怪物行为或规则提示（最低功能已在 P2/P3 前置实现，P7 只负责文案与演出打磨）。

---

## 已排除项（第一阶段明确不做）

完整延后项、延后原因和回归条件见 [`00-next-stage-expansions.md`](00-next-stage-expansions.md)。本节只列第一阶段明确不做的内容边界。

- 第二个副本主题或怪物
- 原形之间的相性或组合系统
- 基地设施升级
- 多结局系统
- 研究树
- 拟人化亲密互动内容（100% 后内容，第一阶段只需要基础好感提示）
- 程序化连通走廊（三层地图结构的第三层，第三阶段再评估）
- 完整高价值最终美术与全套角色 Live 动画（垂直切片先使用占位/局部高价值演示）
- Live2D Cubism / Spine 等外部动画运行时（第二阶段后按插件采纳门槛评估）
